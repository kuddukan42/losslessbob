09Redesign Notes

Recurring patterns to lock in

Seven decisions inform every screen above. Calling them out here so we can debate them once and apply consistently rather than re-deciding per-tab. The first one (Scale) is new in v3 and is the most important.

0 · Scale — table patterns for thousands of rows

Search hits 16,630 rows. My Collection hits 15,967. Pagination at this scale forces you through hundreds of page-clicks; a flat scroll without paging makes the browser choke. The system used on every table in this deck is the same recipe:

The recipe

1. Virtualized rendering. Only the visible rows are in the DOM. Scroll renders the next batch. Works the same at 100, 1,000, or 100,000 rows.
2. Sticky header. Column titles never leave the top of the table area, no matter how far you scroll.
3. Sticky first column. LB# (or Folder) stays anchored when you scroll horizontally on wide tables.
4. Faceted filters with live counts. Each filter chip shows "matches N rows" so narrowing is informed, not blind.
5. Result summary, not page numbers. "245 of 16,630 shown · Decade: 80s · Public" replaces "Page 1 of 333".
6. Saved views. Star a filter combination, name it ("A-rated unowned"), come back to it in one click.
7. Group-by collapsible sections. Group by year / decade / location / status; click a group header to fold it.
8. Type-ahead jump. Type into any column header to jump to the first match.
9. Density picker. Compact mode shows ~55 rows per viewport; Comfortable ~25. Lives in Settings, applies globally.
10. Keyboard nav. ↑/↓ to move row, Space to select, Enter to open, ⌘/Ctrl+F for inline find.

Bulk operations (Pipeline-specific)

11. Multi-select with checkboxes + shift-click ranges + ⌘/Ctrl+A select-all-filtered.
12. Counted filter chips become bulk targets. Click "32 ready" then "Apply all 32 proposed renames" \u2014 one click commits the batch.
13. Live progress banner during runs: "Verifying 47 / 82 \u00b7 12 passed \u00b7 3 mismatched \u00b7 32 remaining".

A · Row status — keep the tint, theme-safe it

You said the green/pink/yellow row tinting is useful, so we're keeping it. But to survive any accent color (Plum, Rust, Crimson, etc.), the tint becomes a much more desaturated wash that always sits on the neutral background — not on top of the accent. The status pill in the dedicated column carries the precise label, and a 3px left-edge bar lets you scan a column even when the wash is subtle.

Three layers, redundant on purpose

1. Subtle row wash (the part you like).
2. 3px left-edge bar (for screen-readers / colorblind scanning).
3. Outlined status pill in a dedicated column (precise label).

B · Vibe: GitHub Desktop

For the hi-fi pass: friendly, not clinical. Light surfaces, generous padding, soft 1px borders instead of heavy chrome, sans-serif type at comfortable sizes (12-13pt body, 11pt mono for numbers and paths). The accent color paints buttons, the active sidebar row, focus rings, and selection — nothing else. Tables get breathing room by default; you toggle Compact in Settings if you need maximum density.

What this rules out

✕ Heavy purple chrome and full-saturation backgrounds.
✕ Tab rows with bordered tab shapes (replaced by sidebar nav and filter chips).
✕ The 20-color theme editor as the default surface (moved behind "Custom…").

C · Theme = Mode × Accent

Mode: Light, Dark, System. Accent: pick one of eight presets — Indigo, Plum, Rust, Forest, Teal, Amber, Gray, Crimson. Status colors (green / amber / red) are fixed across themes for accessibility and live on the row-status bar + pill, so they never collide with whichever accent you pick.

Density picker, also in Settings

Comfortable · Default · Compact. Drives row height and padding globally. Your saved column widths still persist via the existing "Save column widths" control.

D · Curator mode — a single switch

The biggest UI win for a brand-new collector. By default the sidebar shows: Home, Ingest, Library, Media, Settings. That's it. Flip Curator mode in Settings → the sidebar reveals DB Editor, Scraper, Publish Master Update, Geocoding tools, and "Reset Database" promotes its danger styling.

On first launch we can detect whether you're a curator (have you ever clicked Publish? does Curator mode flag exist in the DB?) and skip the toggle. New users never see options they shouldn't touch.

E · One folder, one queue, every step

The recurring pain point you described: dropping the same folder into Lookup, then Verify, then Rename, then LBDIR. The Pipeline screen plus a shared queue across all four standalone drill-ins means you import a folder once, and it shows up everywhere. Right-click any folder anywhere in the app → Send to pipeline.

Cross-screen actions

Every LB row gets the same right-click menu — Open in Search, Open in Bootlegs, Open Attachments, Open Spectrograms, Reveal on disk, Copy LB#. You don't have to choose the right tab first.

F · Map stays simple

The map opens in a browser today — that's fine, no need to embed Leaflet inside the PyQt app. The Map "screen" stays a filter form that hands off to the browser. Geocoding (the part that actually matters) moves to Curator mode where it belongs — a single button that runs the geocode pass and reports coverage.
