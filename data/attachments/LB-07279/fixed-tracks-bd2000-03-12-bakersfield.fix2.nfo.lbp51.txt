This is LB-5034 (straight from LB-4101), but as far as I still left some issues I didn't care about so much then, while listening to it again couple of nights ago I thought wouldn't be so hard to clean it up and eliminate the remained problems without harming the sound in a particular perceptible way so I fixed those flawed tracks in my previous work and added them to a separate subfolder into the main folder.

Lineage: stereo audience recording transfered by Simon M. > cdr > EAC > WAV > FLAC > wav (flac frontend 1.1.2) > LB-5034 version waves (adobe audition 1.0) > flac (flac frontend 1.1.2 - align on sb option - level 8 - verify on)

Additional lineage for the fixed tracks included in this set: flac (flac frontend 1.1.2) > wav (flac frontend 1.1.3) > fixed wavs (Adobe Audition 1.0) > flac (flac frontend 1.1.3 - align on sb option - level 8 - verify on)

To make things right I decoded d1t03-09 but just worked on t03 and 04 of the 1st disc. Once done I encoded the whole bunch using the align on SB option but there were not errors or smaples to be carried over, so the flacs for t05-09 remained as they were. However md5's didn't match (most likely cause I did it with version 1.1.3, instead 1.1.2 I used for the previous set) while the fingerprints are just the same. I added log file for the recent conversion and flac fingerprints as I did with the previous set so anyone ineterested will be able to verify them. The new flacs for t05-09 were deleted as, even not getting same md5 signatures, they had identical ffp and wavs were an exact eac match to the old ones.
